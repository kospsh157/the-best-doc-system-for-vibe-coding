# 구조

## TL;DR (10줄 이내)
- 현재 폴더 구조와 계층 매핑을 기록한다.  
- 변경 시 즉시 갱신한다.  

## 사용
- 레벨: L0 (항상 읽기)  
- 읽기: 세션 시작 시 또는 구조 변경/리팩터링 시.  
- 갱신: 폴더 구조/경계/계층 매핑이 바뀌었을 때.  
## 디렉터리 구조 (초기 제안)
- `src/domain/` : 엔티티, 값 객체, 도메인 서비스  
- `src/application/` : 유스케이스, 애플리케이션 서비스  
- `src/interface/` : 컨트롤러, 프레젠터, DTO  
- `src/infrastructure/` : DB, 외부 API, 메시징 등 구현  
- `src/entrypoints/` : HTTP/CLI 등 진입 지점  
- `tests/` : 테스트  
- `docs/` : 문서  

## 경계 체크
- 도메인에서 외부 의존성 참조 금지  
- 인터페이스/인프라 구현은 포트 기반으로 연결
