# 로드맵

## TL;DR (10줄 이내)
- 로드맵의 기준은 "AI가 다음 세션에서 바로 이어서 작업 가능한 상태"다.  
- 주제 확정 전에는 구조/문서/검사 체계 완성도를 우선한다.  

## 사용
- 레벨: L1 (작업 재개/우선순위 확인 시)  
- 읽기: 마일스톤/우선순위 확인이 필요할 때.  
- 갱신: 목표/우선순위/리스크가 변경되었을 때.  

## 마일스톤
- M1: 세션 복귀 문서 체계 안정화(L0/L1/L2, 로그/체크리스트, 복귀 규칙)
- M2: 프로젝트 주제 확정 및 첫 기능 명세(`docs/specs/feature-*.md`) 작성
- M3: 명세 기반 첫 유스케이스/어댑터 구현 + 테스트 추가

## 우선순위/리스크
- 우선순위: 주제 확정 > 기능 명세 작성 > 구현
- 리스크: 주제 미정 상태가 길어질수록 문서/코드 분리 가능성 증가
- 대응: `docs/session/README.md`의 "다음 작업 Top 3"를 세션마다 최신화
