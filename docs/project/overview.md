# 프로젝트 개요

## TL;DR (10줄 이내)
- 이 프로젝트의 1차 목적은 AI 에이전트의 세션 복귀 속도를 높이는 문서/구조 시스템 제공이다.  
- 주제(도메인) 미정 단계에서도 Clean Architecture 골격을 먼저 고정한다.  
- 세션마다 문서만 읽어도 현재 상태/다음 작업/리스크를 즉시 파악할 수 있어야 한다.  

## 사용
- 레벨: L0 (항상 읽기)  
- 읽기: 세션 시작 시 또는 프로젝트 이해가 필요한 경우.  
- 갱신: 목표/비목표/시나리오가 변경되었을 때.  

## 30초 요약 (AI 우선)
- 무엇을 하는가: 세션 재시작 친화적 문서 시스템 + 클린아키텍처 시작 골격
- 왜 필요한가: 컨텍스트 손실 없이 다음 작업으로 즉시 복귀하기 위해
- 지금 상태는 어떤가: 구조/문서/검사 장치 준비 완료, 비즈니스 주제는 미정

## 항상 알아야 할 것 (필수)
- 구조 규칙: `docs/architecture/structure.md`
- 현재 상태: `docs/session/README.md`
- 지난 작업 기록: `docs/session/LOG.md` 최신 1~3개
- 구현 전후 점검: `python3 scripts/check_arch.py`

## 문제/배경
- 세션이 바뀔 때마다 AI 컨텍스트가 초기화되어 프로젝트 이해 비용이 반복된다.
- 주제가 확정되기 전에는 코드보다 구조가 먼저 흔들리기 쉬워 유지보수성이 떨어진다.

## 목표
- 문서만으로 프로젝트의 목적/상태/다음 작업을 빠르게 복원한다.
- Clean Architecture 경계를 기본값으로 고정한다.
- 구조 위반을 자동 검사해 유지보수 비용을 낮춘다.

## 비목표
- 특정 비즈니스 도메인 구현을 지금 단계에서 확정하지 않는다.
- 프레임워크/언어 종속적인 상세 구현을 선행하지 않는다.

## 핵심 사용자/사용 시나리오
- 사용자: AI 에이전트(주), 개발자(부)
- 시나리오: 새 세션 시작 → L0 문서 읽기 → 현재 상태/다음 작업 파악 → 작업 재개

## 성공 기준
- 세션 시작 후 60초 내에 현재 목표/다음 작업/리스크 파악 가능
- 구조 변경 시 문서와 검사 규칙이 동기화됨
- 신규 기능 시작 시 `docs/specs/feature-*.md` 기준으로 작업 가능
