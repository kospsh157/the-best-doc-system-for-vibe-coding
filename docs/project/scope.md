# 범위/요구사항

## TL;DR (10줄 이내)
- 현재 범위는 "AI 세션 복귀에 강한 문서 체계 + 클린아키텍처 시작 구조"다.  
- 프로젝트 주제가 확정되기 전까지 비즈니스 구현은 범위 밖이다.  
- 구조/문서/검사 자동화의 일관성을 우선한다.  

## 사용
- 레벨: L0 (항상 읽기)  
- 읽기: 세션 시작 시 또는 범위 확인이 필요한 경우.  
- 갱신: 범위/비범위/제약/성공 기준이 변경되었을 때.  

## 범위
- L0/L1/L2 문서 체계 유지 및 세션 복귀 최적화
- Clean Architecture 디렉터리 구조(`src/*`) 고정
- 계층 의존성 검사(`scripts/check_arch.py`) 운영
- 테스트 구조 골격(`tests/unit`, `tests/integration`) 유지

## 비범위
- 특정 도메인 로직/기능 구현 확정
- 제품 UI/외부 API 계약의 상세 설계 확정
- 배포 인프라/운영 자동화의 고도화

## 제약 조건
- 문서는 `docs/` 아래만 관리
- 구조 변경 시 `docs/architecture/structure.md` 즉시 갱신
- 세션 상태 변경 시 `docs/session/README.md` 최신화
- 구현 전후 `python3 scripts/check_arch.py` 실행

## 주의사항 (항상 확인)
- 주제 미정 상태에서 임의 도메인 가정으로 코드/명세를 확정하지 않는다.
- 문서가 비어 있는 항목은 추정으로 채우지 않고 "미정"으로 명시한다.
- 세션 로그는 최신 1~3개 확인 후 작업을 시작한다.

## 성공 기준
- 새 세션에서 L0 문서만으로 즉시 작업 복귀 가능
- 구조 위반 없이 기능 구현을 확장할 수 있음
- 지난 세션 변경/결정/다음 작업이 문서에서 끊기지 않음
