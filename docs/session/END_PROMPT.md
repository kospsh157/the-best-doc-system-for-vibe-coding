# 세션 종료 프롬프트

이 파일의 프롬프트를 세션 종료 시 사용자 입력으로 사용한다.

## 사용
- 레벨: L0-도구 (세션 종료 시 사용)  
- 읽기: 세션 종료 시.  
- 갱신: 종료 절차/요약 규칙이 변경되었을 때.  

## 사용자 종료 문구 예시
- `세션 종료할꺼야 문서 체계에 맞춰서 지금까지 한 작업들 문서 업데이트 해줘`
- 위 문구가 들어오면 아래 PROMPT 절차를 그대로 수행한다.

## PROMPT
```
세션 종료. 아래 절차를 반드시 수행하라.
1) docs/session/README.md 업데이트: 프로젝트 스냅샷/현재 상태를 최신화하고, 없으면 "없음"으로 명시.
2) docs/session/LOG.md에 새 항목 추가(최신 상단). 날짜/시간, 목표/변경/파일/참조/결정/테스트/다음/재개 포인트 항목을 채운다. 참조는 세션 중 읽은 모든 문서(L0 포함).
3) 이번 세션에 영향이 있는 문서(L1/L2/L3)를 갱신한다. 변경이 없으면 "변경 없음"으로 기록.
4) 토큰 상태를 확인한다: `python3 scripts/doc_token_guard.py`.
   - 30000 이상이면 비파괴 간단 정리(중복 제거/링크화/구조 개선/요약 보강)를 수행한다.
   - 40000 이상이면 전체 문서를 정밀 검토해 삭제/축약 후보와 영향/절감량을 제시한다.
5) 비파괴 정리(중복 제거/링크화/구조 개선/요약 보강)는 승인 없이 수행한다.
6) 사용자 승인 없이 문서 원문 삭제/축약을 수행하지 않는다. 승인 후 적용 내역은 LOG에 남긴다.
   - `프로젝트 기초`, `핵심 설계`, `지켜야 할 지침` 섹션은 삭제/축약하지 않는다.
7) `python3 scripts/check_docs.py`를 실행해 문서 품질(중복/누락/플레이스홀더)을 점검하고 실패 시 수정한다.
8) 응답에는 변경된 파일 목록과 테스트 실행 여부를 요약한다.
```
