# 기능 명세

## TL;DR (10줄 이내)
- 작업 중인 기능만 읽는다.  
- 각 기능은 `docs/specs/feature-이름.md`로 분리한다.  
- 명세는 AI가 다음 세션에 바로 재개할 수 있도록 작성한다.  

## 사용
- 레벨: L2 (작업 영역별)  
- 읽기: 기능 추가/변경, 관련 버그 분석 시.  
- 갱신: 새 기능 명세 작성 또는 공통 항목 변경 시.  
## 인덱스
- 없음 (프로젝트 주제 미정)  

## 작성 규칙
- 기능 단위로 섹션을 분리한다.  
- 배경, 플로우, 에러/경계 조건을 명시한다.  
- 관련 파일/모듈을 기록한다.  
- "현재 상태(완료/미완료)"와 "다음 액션 1개"를 명시한다.  
- 구현 중 결정사항은 필요 시 `docs/architecture/decisions.md`에 연결한다.  

## 템플릿
- `docs/specs/TEMPLATE.md` 사용  
## 사용 규칙
- 새 기능 명세는 반드시 템플릿으로 작성한다.  
- 여러 명세에 반복되는 항목이 생기면 템플릿에 추가한다.  
- 기능 동작에 영향이 없는 변경은 명세를 수정하지 않는다.  
- 세션 종료 전, 진행 중 기능 명세의 "재개 포인트"를 최신화한다.
