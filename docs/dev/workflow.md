# 개발 워크플로우

## TL;DR (10줄 이내)
- 개발 절차와 품질 기준만 기록한다.  

## 사용
- 레벨: L2 (작업 영역별)  
- 읽기: 개발 절차/품질 기준 확인이 필요할 때.  
- 갱신: 워크플로우나 품질 기준이 변경되었을 때.  
## 워크플로우
1) 요구사항 확인  
2) 설계/명세 갱신  
3) (최초 1회) 훅 설치 `./scripts/install_git_hook.sh`  
4) 구현  
5) 아키텍처 경계 검사(`python3 scripts/check_arch.py`)  
6) 테스트  
7) 문서 갱신  
8) 토큰 상태 점검(`python3 scripts/doc_token_guard.py`)  
9) 30000 이상이면 비파괴 간단 정리, 40000 이상이면 정밀 검토 + 사용자 승인 후 삭제/축약 반영  
10) 문서 품질 검사(`python3 scripts/check_docs.py`)  

## 품질 기준(초기)
- 테스트 가능한 구조 유지  
- 계층 경계 침범 금지  
- 문서 동기화 유지  
- 아키텍처 검사 통과 필수  
- pre-commit 훅으로 검사 자동 실행 권장  
- 문서 검사 통과 필수  
- 사용자 승인 없는 문서 삭제/축약 금지  
- `프로젝트 기초/핵심 설계/지침` 섹션 삭제/축약 금지
